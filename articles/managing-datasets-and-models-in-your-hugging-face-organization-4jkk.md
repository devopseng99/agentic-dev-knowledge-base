# Managing datasets and models in your Hugging Face organization
**Author:** Julien Simon  **Published:** December 3, 2021

## Overview
This article presents a video tutorial demonstrating how to manage ML assets within a Hugging Face organization. The tutorial covers foundational organizational tasks including setting up organizations, creating private repositories via command-line tools, importing existing models and datasets using version control, customizing dataset and model documentation cards, and controlling member access levels.

## Key Concepts
- Organization creation on Hugging Face
- Private repository management using CLI
- Git-based model and dataset imports
- Dataset and model card editing
- Permission management for team members

## Topics Covered
- Creating Hugging Face organizations for team collaboration
- Setting up private model repositories using the CLI
- Importing pre-existing models and datasets into your organization
- Creating informative model cards for documentation
- Managing team member permissions and access levels
- Using version control workflows with Hugging Face

## Use Cases
Managing models and datasets within an organization enables:
- Centralized model governance
- Controlled access to proprietary models
- Collaboration across team members
- Versioned model and dataset history
- Automated CI/CD pipelines for model updates

Referenced datasets and models in this tutorial:
- Dataset: Reuters21578 corpus
- Model: AutoNLP Reuters Summarization
