# AI Regulation Reference: A Structured Database of AI Compliance Obligations Across Jurisdictions
**Author:** snapsynapse  **Published:** April 7, 2026

## Overview
Introduces EveryAILaw.com, a free reference tool for understanding AI regulatory obligations across multiple jurisdictions. Uses an "obligation-first ontology" where compliance concepts serve as the organizing principle rather than individual regulations.

## Key Concepts

**Coverage**
- 51 regulations tracked across 8 regulatory authorities
- 10 core obligations: transparency, human oversight, risk assessment, bias prevention, and more
- Three access methods: HTML browsing, JSON API, MCP server for AI assistants

**When to Use It**
- Checking applicable AI regulations across regions
- Comparing obligations across jurisdictions
- Tracking enforcement timelines
- Developer workflow integration via JSON API

**When NOT to Use It**
Functions as reference material, not legal counsel. Organizations requiring binding compliance assessments must consult attorneys. Current coverage emphasizes U.S. state regulations and the EU AI Act.

**Core Design Principle**
Fundamental compliance concepts remain more stable than the regulatory landscape itself — an obligation-first approach that outlasts regulatory churn.

**Access**
- Live site: everyailaw.com
- Built on Knowledge-as-Code template
