# Recognize Governance And Compliance Regulations For AI Systems
**Author:** Ntombizakhona Mabaso  **Published:** January 26, 2026

## Overview
AWS AI Practitioner exam content covering how organizations demonstrate their AI systems are controlled, auditable, and compliant. Covers recognizing compliance standards and AWS services supporting governance.

## Key Concepts

**Regulatory Compliance Standards**
- ISO: Broad standards for security and quality management controls
- SOC (1/2/3): Audit reports providing assurance about organizational controls for security, availability, confidentiality, processing integrity, and privacy
- Algorithm Accountability Laws: Emerging regulations requiring transparency, risk management, auditing, and responsible use of automated decision systems

**AWS Services for Governance and Compliance**
- AWS Config: Tracks resource configurations against compliance rules for drift detection
- Amazon Inspector: Identifies vulnerabilities in EC2 and container workloads
- AWS Audit Manager: Collects evidence and maps controls to compliance frameworks
- AWS Artifact: Central repository for compliance reports and agreements (SOC, ISO)
- AWS CloudTrail: Records API activities for auditing and incident investigation
- AWS Trusted Advisor: Provides security recommendations supporting governance goals

**Data Governance Strategies**
1. Data Lifecycle Management: Define collection, storage, usage, sharing, archival, deletion
2. Logging: Record data access and model endpoint calls for audit support
3. Residency: Ensure data remains in required geographic locations
4. Monitoring/Observation: Monitor policy violations and abnormal access patterns
5. Retention: Maintain required data/logs, then safely dispose of sensitive information

**Governance Processes**
- Policies: Written rules for acceptable use, data handling, and incident response
- Review Cadence: Scheduled reviews of models, prompts, datasets, and permissions
- Review Strategies: Human review for high-risk outputs, red-teaming, documented approvals
- Transparency Standards: Model documentation including model cards and user disclosures
