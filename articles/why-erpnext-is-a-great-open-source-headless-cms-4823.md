# Why ERPNext is a Great Open Source Headless CMS
**Author:** grynn-in  **Published:** July 10, 2021

## Overview
Among approximately 80 headless CMS options on Jamstack.org, most have poor filtering systems requiring multiple requests for complex queries. ERPNext offers a compelling alternative combining headless CMS capabilities with enterprise ERP features.

## Key Concepts

**CMS Classification Categories**
- API-Driven Systems: Most headless CMSs have API access but "terrible filtering systems"
- Established Core Products: e.g., PIMCORE for product information management
- Open Source vs. Closed Source: About half the available options are open source

**Why ERPNext as a Headless CMS**
- "800 odd REST API endpoints with filtering options"
- Built-in accounting, stock management, and manufacturing
- 50,000+ deployments globally
- Socket.io and Redis cache support
- GraphQL compatibility through community extensions

**Key Technical Advantages**
- Rich filtering across all 800+ endpoints (unlike most headless CMSs)
- Real-time capabilities via Socket.io
- Built-in Redis caching
- Multi-tenant support
- Full ERP functionality alongside content management

**Target Audience**
Particularly suited for SMEs and startups requiring accounting systems alongside content management without extensive development budgets.
