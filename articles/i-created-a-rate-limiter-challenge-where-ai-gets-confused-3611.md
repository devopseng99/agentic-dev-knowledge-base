# I Created a Rate Limiter Challenge Where AI Gets Confused

**Author:** Md Mijanur Molla
**Published:** March 21, 2026

## Overview
Rate limiting appears deceptively simple but AI-generated solutions struggle with real-world edge cases. A coding challenge on VibeCode Arena tests whether developers and AI systems truly understand rate limiting complexity.

## Key Concepts

### Critical Issues with Naive Implementations
- Time handling issues
- Reset logic problems
- Lack of scalability for multiple users
- Absence of concurrent request support

### AI Performance Observations
- Some AI models produce basic functional code
- Others overlook real-world edge cases
- Many disregard scalability requirements
- Few actually "think like a real system"

### Key Insight
"Rate limiting looks simple. But real systems are never simple. The difference between 'working code' and 'production-ready system' is where real engineering starts."
