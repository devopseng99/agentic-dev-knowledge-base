# The AI Coding Workflow That Actually Works: Separate Planning from Execution

**Author:** Matthew Hou
**Published:** February 23, 2026

## Overview

Describes an approach to AI-assisted coding that separates planning from execution. The core argument: asking AI to simultaneously figure out requirements and write code produces suboptimal results.

## Key Concepts

### Proposed Workflow

1. Write a plain English specification (not code)
2. Request AI create a task breakdown without implementation
3. Review and refine the plan (~2 minutes)
4. Execute each subtask individually
5. Handle integration testing personally

### Supporting Claims

- Planning-first approach yields 40% faster end-to-end results despite feeling slower
- Early assumption-catching prevents costly debugging
- Maintains architectural consistency
- Improves code quality through smaller scopes

### Limitations

The methodology does not replace bug investigation, architecture decisions, or code review responsibilities. Developer judgment remains essential.
