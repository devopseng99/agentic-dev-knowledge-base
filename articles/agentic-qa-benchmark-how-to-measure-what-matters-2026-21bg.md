# Agentic QA Benchmark: How to Measure What Matters (2026)
**Author:** Shiplight  **Published:** April 13, 2026

## Overview
Addresses evaluation challenges for agentic QA platforms, arguing that traditional benchmarks fail to capture long-term performance. Rather than measuring one-time capabilities, proposes assessing how platforms sustain value as applications evolve over months.

## Key Concepts
1. **Self-Heal Rate Under Real UI Change** — Percentage of test failures from UI modifications that resolve automatically
2. **CI Pass Rate Stability (90 days)** — Whether test suites require ongoing manual maintenance
3. **Coverage Growth Velocity** — Speed of expanding test coverage relative to engineering hours invested
4. **Maintenance Hours Per Week** — Engineering time spent on test upkeep and broken test resolution
5. **Mean Time to Resolution (MTTR)** — Duration from test failure to diagnosis and fix

"The benchmark question is not 'what can it do?' but 'what does it do to your test suite over 90 days?'"

Recommends a structured 4-week PoC framework tracking baseline metrics, platform onboarding, UI change testing, and steady-state performance.
