# What Does a Good Hackathon Submission Look Like?
**Author:** DoraHacks
**Published:** January 31, 2026

## Overview
A guide from DoraHacks on crafting strong hackathon submissions, featuring the 3W1H model (What, Why, Who, How) and a case study of ForestGuard Agent, an AWS hackathon winner.

## Key Concepts
1. **Read Requirements Carefully** - Prevent disqualification on technicalities
2. **Show Work, Not Hype** - Use the 3W1H model; avoid vague marketing language
3. **Highlight Differentiation** - Ensure judges grasp unique value quickly
4. **Make Eligibility Obvious** - Include screenshots, code snippets showing required technology
5. **Verify All Links Function** - Test repositories, demos, and contacts before submission

Case Study: ForestGuard Agent detects, verifies, and reports deforestation events in real time using satellite imagery, drone data, and community photos.
