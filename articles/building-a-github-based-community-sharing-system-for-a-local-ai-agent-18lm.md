# Building a GitHub-Based Community Sharing System for a Local AI Agent
**Author:** Kim Namhyun
**Published:** March 8, 2026

## Overview
Design and implementation of a GitHub PR-based community sharing system for Xoul, a locally-running AI agent. Covers repository structure, code standardization, and the decision to use server-side GitHub PAT proxy instead of OAuth.

## Key Concepts

### Architecture: Server as GitHub Proxy

```plaintext
User Desktop -> Server /api/share -> GitHub API (Server PAT) -> PR created
     ^                                                            |
   PR URL returned <---------------------------------------- PR URL
```

### Repository Structure

```plaintext
xoul-store/
├── codes/
│   ├── finance/binance-portfolio.py
│   ├── games/arena-agent-v1.py
│   └── manifest.json
├── personas/
│   ├── research/p-001-en.md
│   └── manifest.json

│   └── manifest.json
├── dist/          <- Auto-generated by CI
│   ├── codes.json
│   └── personas.json
└── .github/workflows/build.yml
```

### used_by References Pattern

```python
# codes table
used_by TEXT DEFAULT '[]'   # e.g., ["Tech Trend Research", "Daily Blog"]
```

### Code Name References (Eliminating Inline Duplication)

```json
{"type": "code", "code_name": "crypto prices"}
```

Replaces inline duplication:
```json
{"type": "code", "content": "import urllib.request\n...200 lines..."}
```

### def run() Standardization

```python
# Before (flat script)
import urllib.request, json
url = f"https://api.coingecko.com/...?ids={coins}"

# After (standardized -- LLM reads signature + docstring)
def run(coins: str = "bitcoin,ethereum"):
    """
    coins: Coin IDs (default: bitcoin,ethereum)
    """
    import urllib.request, json
    url = f"https://api.coingecko.com/...?ids={coins}"
```

### Design Principle
"fail-fast > silent wrong behavior" -- Missing config triggers RuntimeError immediately rather than silently loading incorrect models.
