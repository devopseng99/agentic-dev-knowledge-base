# Google's Project Jitro Just Redefined What a Coding Agent Is

**Author:** Om Shree
**Published:** April 22, 2026

## Overview

Contrasts Google's emerging Project Jitro with existing coding agents. The core distinction: "Project Jules used to tell your AI what to do. Jitro tells it what you want." Jitro shifts toward high-level goal-setting rather than prompt-driven task execution.

## Key Concepts

### The Problem with Current Coding Agents

Tools like GitHub Copilot, Cursor, Windsurf, and OpenAI's Codex operate on a prompt-response model where developers remain project managers and QA reviewers. This fails for objective-based goals like reducing memory leaks by 20% or achieving 100% accessibility scores.

### How Project Jitro Works

- **Workspace Model:** A persistent collaborative environment (not one-shot)
- **API Operations:** Goal listing/creation, insight tracking, tool integration configuration
- **Integration:** Model Context Protocol (MCP) remote servers and API connections
- **Transparency:** Surfaces reasoning when making technical decisions

### Primary Use Cases

Engineering teams managing large codebases benefit most, particularly for performance optimization, test coverage improvement, and accessibility compliance.

### Strategic Significance

Shift from measuring productivity by prompt quality to measuring it by outcome clarity. Developers transition toward guiding AI systems, reviewing outputs, and aligning technical work with business objectives.

Jules V1 demonstrated viability with 140,000+ shared code improvements during beta. Project Jitro remains in pre-launch, expected at Google I/O 2026.
