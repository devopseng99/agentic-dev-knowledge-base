# Engineering the Modern Turing Test: Building BotSpot
**Author:** Harish Kotra  **Published:** May 2, 2026

## Overview
BotSpot is a swipe-based game that challenges users to distinguish between human-written and AI-generated content using Google's Gemini 2.0 Flash model. Explores the technical and design challenges of creating a modern Turing Test application with an engaging user experience.

## Key Concepts
- **Difficulty Strategy**: Explicit prompting to force AI models away from recognizable patterns ("avoid typical AI tropes like 'tapestry'") and toward human-like flaws (typos, irregular pacing, slang)
- **Swipe Mechanics**: Motion-based card interactions using Framer Motion's `useMotionValue` and `useTransform` APIs to map horizontal displacement to visual feedback
- **Procedural Audio**: Web Audio API implementation for generating sound effects dynamically — no pre-recorded files
- **Feedback Loop**: Explanatory `style_note` generated by AI for each piece of content, educating users on classification reasoning
- **Cyber-Brutalism Design**: Action Pink (#FF0080) and Acid Green (#00FFC2) with neubrutalist shadow effects
- **Gemini 2.0 Flash**: Google's model used for content generation with human-mimicking difficulty settings
