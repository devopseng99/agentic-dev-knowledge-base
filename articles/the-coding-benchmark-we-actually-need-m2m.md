# The Coding Benchmark We Actually Need
**Author:** Mixture of Experts  **Published:** May 7, 2026

## Overview
Critiques ProgramBench, a coding agent evaluation framework, arguing it measures memorization rather than practical capability. Proposes redesigning benchmarks to assess real-world value generation instead.

## Key Concepts
1. **ProgramBench's Limitation** — Forbids web access, decompilation tools, and source code, forcing agents to reconstruct programs solely from training data recall rather than actual problem-solving
2. **Memorization vs. Capability** — Current testing strips away tools real engineers use; high performance indicates training set memorization, not genuine engineering ability
3. **Free-Form Input Value** — ProgramBench's strength: requires unstructured implementation without predefined scaffolding
4. **Outcome-Based Scoring** — Proposed alternative: evaluate agents on revenue generation, customer satisfaction, and long-horizon coherence rather than test pass rates
5. **Hybrid Benchmark Proposal** — Combine free-form inputs with real development tools, genuine outcomes, and business metrics to measure "GDP value generated"
