# I Built an AI SDLC Scaffold
**Author:** Claudio Felicioli  **Published:** March 16, 2026

## Overview
Felicioli created an open-source repository template designed to integrate AI agents (particularly Claude Code) into software development workflows. The scaffold emphasizes pre-coding phases: objectives elicitation, user story creation, requirements definition, and architectural design.

## Key Concepts
**Core Principles**:
1. **Everything-in-repo**: "objectives, user stories, requirements, architecture, decisions, and task tracking all live alongside the source code"
2. **Context-window efficiency**: Instructions and project knowledge are organized hierarchically so agents can load only necessary information; artifact collections use markdown table indexing
3. **Decision capture**: AI-generated decisions are documented as structured artifacts within the repository, maintaining traceability across sessions

**AI-first development approach** focused on pre-coding phases

**Hierarchical knowledge organization** for efficient token usage

**Persistent decision tracking** ensuring consistency across multiple AI sessions

**Repository-as-documentation** model where the repo itself serves as the living project document
