# The 5 Stages of Using Amazon Q Developer CLI Agent
**Author:** Brooke Jamieson
**Published:** April 9, 2025

## Overview
A developer-psychology-driven exploration of adopting the Amazon Q Developer CLI agent, mapping the emotional journey from denial to acceptance. Covers the agent's capabilities including multi-step reasoning with Claude 3.7 Sonnet, tool integration, and real-world productivity gains.

## Key Concepts

### The Five Stages

1. **Denial:** "I don't need AI help. My terminal skills are flawless."
2. **Anger:** "Who needs an AI that runs Git commands?"
3. **Bargaining:** "Fine, I'll just use it for AWS CLI commands I can't remember..."
4. **Depression:** "I've spent how many hours typing 'aws s3 ls' when I could've been learning guitar?"
5. **Acceptance:** "Q, while I grab coffee, could you scaffold a serverless app, configure API Gateway, and write unit tests?"

### Agent Capabilities
- Uses Anthropic's Claude 3.7 Sonnet for multi-step reasoning
- Runs commands using tools already installed on your system
- Engages in multi-turn conversations for complex tasks
- Accesses and modifies files in local environment
- Queries and interacts with AWS resources

### Real-World Results
A developer finished a task expected to take three days in less than an hour, including writing tests and getting it running.

### Emoji-Driven Development Experiment
Prompted with: "Create a Python web scraper that does: web -> parse -> search -> chart -> save"
The agent understood intent from emoji sequences and generated a complete web scraper following the exact workflow.
