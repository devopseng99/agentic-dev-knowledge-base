# Don't ask AI to build the house. Ask it to build the scaffolding.
**Author:** Raffaele Pizzari  **Published:** April 1, 2026

## Overview
The article argues that AI's greatest value lies not in generating production code, but in creating temporary support tools that aid development. Rather than asking AI to architect complete systems, developers should leverage it to build "jigs" — disposable tools that provide visual feedback and validate complex logic without long-term maintenance burdens.

## Key Concepts
1. **The Scaffolding Metaphor** — Use AI for temporary support structures, not permanent solutions
2. **Jigs in Software** — Temporary tools (borrowed from woodworking) that guide and validate work
3. **Disposable Code** — Code designed to be thrown away after serving its purpose
4. **The "Dirty Prompt"** — Explicitly instructing AI to ignore best practices (clean architecture, optimization) in favor of quick functionality
5. **ROI of Temporary Tools** — AI collapses code generation costs, making short-lived utilities economically viable
6. **Visual Feedback Loops** — Using AI-generated visualizations to stress-test algorithms before production implementation

The key insight: stop asking AI to build the cathedral. Ask it to build the scaffolding so you can climb up and do the work yourself, safely.
