# HMP as an Implementation of the Application Layer in ANP
**Author:** kagvi13
**Published:** January 29, 2026

## Overview
How HMP (HyperCortex Mesh Protocol) functions as a reference implementation within ANP's Application Layer, providing memory, dialogue continuity, and semantic navigation.

## Key Concepts

### ANP Layer Architecture
- **Layer 1:** Identity & Encryption
- **Layer 2:** Meta-Protocol negotiation
- **Layer 3:** Application (where HMP operates)

### HMP's Role
HMP focuses on what should be transmitted so meaning and context persist over time, using proof-chains, timestamps, and resonance scoring.

### Communication Flow
1. ANP Layer 1 verifies identity
2. Layer 2 negotiates protocol use
3. HMP Layer 3 transmits semantic content with contextual proof-chains

### Key Advantages
- HMP avoids reinventing DID, encryption, or peer discovery by leveraging ANP
- ANP gains concrete Layer 3 implementation without vendor lock-in
- Both protocols operate independently or complementarily
