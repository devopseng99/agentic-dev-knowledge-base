# Planning for a Project: Boilerplate
**Author:** Sadiqur Rahman  **Published:** June 28, 2020

## Overview
The article advocates for documenting project plans before coding begins. Writing down planning details prevents feeling overwhelmed during development and keeps the mind free by externalizing information.

## Key Concepts
**Frontend Stack**: HTML, SCSS (preferred over CSS for reusability), and JavaScript or frameworks

**Backend Framework**: Laravel, Codeigniter, Node.js, or Django; TypeScript recommended over JavaScript for Node.js

**Database Selection**:
- SQL (MySQL, PostgreSQL, MariaDB) for relational data
- NoSQL (MongoDB, ArangoDB, DynamoDB) for schema flexibility and large datasets
- Redis or memcached for caching frequently-queried, stable data

**External Services**:
- APIs: Facebook/Google authentication, Dialogflow, Twilio
- Email: SendGrid (recommended) or Nodemailer
- Testing: Jest, Jasmine, Mocha, Chai for Test Driven Development

**DevOps Components**:
- Server: DigitalOcean or AWS EC2
- Web server: Nginx or Apache
- Process manager: PM2
- SSL: Let's Encrypt (free) or commercial providers
- Version control: Git with GitHub/GitLab/Bitbucket
- CI/CD: Jenkins or CircleCI

**Project Management**: Agile methodology with Trello and Slack; code review sessions recommended
